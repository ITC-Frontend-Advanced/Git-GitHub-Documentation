# Git vs GitHub

{% columns %}
{% column width="50%" %}
### Git

<sup><sub>GitHub is an<sub></sup> <sup><sub> </sup><sup><sub>**online platform**<sub></sup> <sup><sub> </sup><sup><sub>built on top of Git.<sub></sup>

It allows you to:

* Track changes in your code
* Create commits and branches
* Move between versions safely

{% hint style="success" %}
<sup><sub>Git works<sub></sup> <sup><sub> </sup><sup><sub>**locally**<sub></sup> <sup><sub> </sup><sup><sub>and does not require the internet.<sub></sup>
{% endhint %}
{% endcolumn %}

{% column width="50%" %}
### GitHub

<sup><sub>Git is a<sub></sup> <sup><sub> </sup><sup><sub>**version control tool**<sub></sup> <sup><sub> </sup><sup><sub>that runs on your computer.<sub></sup>

It allows teams to:

* Store repositories online
* Collaborate with others
* Review code using pull requests

{% hint style="warning" %}
<sup>GitHub requires an internet connection.</sup>
{% endhint %}
{% endcolumn %}
{% endcolumns %}

### Key differences

| Git                  | GitHub                |
| -------------------- | --------------------- |
| Tool                 | Platform              |
| Runs locally         | Runs online           |
| Manages code history | Manages collaboration |
| Works offline        | Requires internet     |

<img src="../.gitbook/assets/file.excalidraw (1).svg" alt="Git manages versions locally, GitHub enables collaboration online." class="gitbook-drawing">

<details>

<summary>How they work together</summary>

* You use **Git** to manage code locally
* You use **GitHub** to share and collaborate

</details>
